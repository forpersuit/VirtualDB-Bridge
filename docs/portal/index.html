VirtualDB-Bridge is a high-performance SQL virtualization gateway. It provides isolated virtual database spaces for each tenant through transparent SQL rewriting, integrated with Cloudflare Access and Email Proxy. VirtualDB-Bridge 是一个高性能的 SQL 虚拟化网关。它通过透明的 SQL 重写技术，为每个租户提供完全隔离的虚拟库空间，同时集成了 Cloudflare 边缘级安全防护与邮件代发服务。

Security Architecture & Zero Trust 安全架构与零信任原则

1. The "Double-Gate" Protocol

1. “双重门禁”协议

Security is not a single wall, but a series of checkpoints. We combine Edge Infrastructure (Cloudflare) with Business Logic (API Keys).

安全不是一堵墙，而是一系列检查站。我们将边缘基础设施（Cloudflare）与业务逻辑（API Key）紧密结合。

2. Defense in Depth

2. 深度防御

From TLS 1.3 encryption to database-level wildcard permissions, every layer is designed to assume the previous one might fail.

从 TLS 1.3 加密到数据库层面的通配符权限，每一层都旨在防御前一层的潜在失效。

3. Verifiable Identity

3. 可验证的身份

JWT validation ensures the request source is authentic, while the mailbox verification prevents email spoofing.

JWT 校验确保请求来源真实，而邮箱验证则彻底杜绝了邮件伪造的可能性。

Database Permission Pattern 数据库权限配置

Create a restricted user in MariaDB/MySQL. This user should only have permissions on databases that start with the tenant's prefix. 在 MariaDB/MySQL 中创建一个受限用户。该用户应仅拥有对以租户前缀开头的数据库的操作权限。

  • Prevents tenants from accessing each other's data even if the proxy fails.
  • 即便网关失效，也能防止租户越权访问其他租户的数据。
  • Uses wildcard \_% to match only the intended prefix.
  • 使用通配符 \_% 来严格匹配指定前缀。
-- Run this for each tenant prefix
GRANT ALL PRIVILEGES ON `tenantA\_%`.* 
TO 'bridge_user'@'%' 
IDENTIFIED BY 'secure_password';
FLUSH PRIVILEGES;

Security Note

安全提示

Never commit your .env file. The gateway uses double-layer auth: Cloudflare Service Token (Header) + API Key (Body).

严禁提交 .env 文件。网关采用双重认证：Cloudflare Service Token (Header) + API Key (Body)。

# .env template
PORT=3002
DB_PASS=secure_password
SMTP_PASS=smtp_password
CF_AUDIENCE_TAG=your_tag
CF_TEAM_DOMAIN=https://xxx.cloudflareaccess.com
