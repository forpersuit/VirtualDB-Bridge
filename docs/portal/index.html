VirtualDB-Bridge 是一个高性能的 SQL 虚拟化网关。它通过透明的 SQL 重写技术，为每个租户提供完全隔离的虚拟库空间，同时集成了 Cloudflare 边缘级安全防护与邮件代发服务。

立即开始部署 了解原理

透明虚拟化原理

graph TD A[开发者/客户端] -->|POST SQL: CREATE DATABASE mydb| B(VirtualDB-Bridge) B -->|身份验证: Cloudflare JWT| C{API Key 校验} C -->|成功| D[SQL 重写引擎] D -->|mydb -> tenantA_mydb| E[(共享数据库集群)] E -->|返回结果| D D -->|抹除前缀| B B -->|JSON 响应| A

SQL 透明重写

自动处理 CREATE, DROP, USE 以及 `db.table` 语法。开发者无需在代码中手动管理复杂的租户前缀。

边缘级安全

利用 Cloudflare Access 拦截非授权请求，双重复合认证确保数据资产万无一失。

统一邮件代理

内置 Nodemailer 代理，支持租户级的发件人校验，简化微服务下的邮件发送逻辑。

三步完成生产部署

01

底层数据库授权

在 MariaDB/MySQL 中创建一个“受限管理员”账户，仅允许其操作特定前缀的库。

-- 允许操作所有以 tenant1_ 开头的库
GRANT ALL PRIVILEGES ON `tenant1\_%`.* 
TO 'bridge_user'@'%' 
IDENTIFIED BY 'secure_password';
FLUSH PRIVILEGES;
02

Cloudflare 配置

在 Zero Trust 控制台中为网关域名添加 Access 策略，并生成 Service Token。

  • 记录 Audience Tag
  • 记录 Team Domain
03

环境变量加载

配置项目根目录下的 .env 文件，填入上述步骤获取的密钥。

PORT=3002
DB_PASS=secure_password
CF_AUDIENCE_TAG=your_tag
CF_TEAM_DOMAIN=https://xxx.cloudflareaccess.com
