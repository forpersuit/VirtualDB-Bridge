Security is Not a Feature,
It's Our Foundation 安全不是插件
而是我们的根基

VirtualDB-Bridge transforms shared databases into secure, virtual workspace clusters. No more cross-tenant leaks. No more complex isolation logic. Just pure, secure SQL. VirtualDB-Bridge 将共享数据库转变为安全的虚拟工作空间集群。彻底杜绝跨租户漏油，告别复杂的隔离逻辑，回归最纯粹、安全的 SQL 开发。

Why it's Secure?

为什么它是安全的？

Imagine a bank where everyone has the same key to the vault, but they only see their own safe deposit box. That's VirtualDB-Bridge.

想象一个银行，每个人都有进入保险库的钥匙，但他们只能看到并打开属于自己的保险箱。这就是 VirtualDB-Bridge 的逻辑。

Encrypted & Untampered

加密且不可篡改

Data travels inside a TLS 1.3 "bulletproof tube." Even if intercepted, it's unreadable. JWT signatures ensure the "package" was never opened.

数据在 TLS 1.3 的“防弹管道”中传输。即使被拦截也无法读取。JWT 签名则保证了“包裹”在途中从未被拆封。

Isolation at Root

根源层隔离

Isolation isn't in your app code (where bugs happen). It's in the Gateway. If an app tries to "cheat," the Gateway rewrites the cheat into a valid local operation.

隔离逻辑不在你的 App 代码中（那里容易出 Bug），而在网关层。如果 App 尝试越权，网关会自动将其重置为合法的本地操作。

Quick Install

# Clone project
git clone https://github.com/forpersuit/VirtualDB-Bridge

# Setup environment
cp .env.example .env
cp tenants.example.json tenants.json

# Run with Docker or Node
npm install && npm run build
npm start
